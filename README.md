[1. Введение](#content)<a id="introduction"></a>
========================================
Данная работа посвящена разработке проекта в области предоставления услуг связанных с творческой деятельности. 

В современном мире существует растет спрос на творческий вид деятельности, появляется всё больше людей которые хотят монетизировать свою творческую деятельность (предоставить услугу творческого направления). Однако отсутвие единой площадки по предоставлению таких услуг осложняет задачу, реализация подобной площадки поможет решить следующие проблемы:

- Поиск партнеров для создания собственного проекта
- Поиск сотрудников для работы в фирме 
- Поиск специалистов для личного заказа

[1.1 Глоссарий](#content)<a id="glossary"></a>
========================================
Мессенджер - функция для обмена мгновенными сообщениями.

Производитель - пользователь который предоставляет свои услуги.

Потребитель - пользователь который пользуется услугами другого пользователя(производителя).

Юзер - это пользователь. Чаще всего имеется в виду пользователь компьютерной программы, сайта, устройства и т.п.

[1.2 Описание предметной области](#content)<a id="region"></a>
========================================

Последние 10 лет в России наблюдается тенденция по поиску работы в интернете. Многие специалисты предпочитают искать сотрудников на различных интернет-площадках из-за удобства данного способа. 

Но именно творческим людям, таким как художники, музыканты, писатели и дизайнеры, действительно тяжело найти работу даже в интернете. Это имеет множество причин:
* Высокая конкуренция;
* Ограниченность мест работы;
* Низкая заработная плата;
* Предвзятость со стороны большинства работодателей в отношении творческих профессий.

В связи с этими факторами, производители не находят потребителя и также в обратную сторону.

Крупные сервисы по поиску работы и работников адаптированы под широкий спектр предоставляемых услуг, по этой причине, чтоб найти команду или одноразовые задачи творческим людям нужно одновременно быть зарегистрированным на множестве площадок и надеяться что они найдут подходящую им работу и их возьмут на её исполнение.

Отсутствие централизованной площадки для предоставления именно творческих услуг - та проблема, которую наш проект предполагает решить.

Соответственно, цель проекта - решение проблемы трудоустройства работников многих творческих специальностей и работодателей нуждающихся в таких кадрах путем создания сервиса, адаптированного под поиск потребителя и производителя данного типа работы.

[1.3 Неформальная постановка задачи](#content)<a id="FSP"></a> 
========================================
Требуется реализовать проект позволяющий взаимодействовать потребителям и производителям творческих услуг, имеющий следующие возможности:

- Наличие удобного и современного каталога объявлений, а так же форму для заполнения объявления производителем. 
- Наличие фильтров для быстрой и удобной навигации по каталогу. 
- Наличие личного кабинета.
- Наличие обратной связи, продукт должен предоставлять возможность потребителю оставлять отзывы и оценки производителю услуг.
- Удобная коммуникация при помощи встроенного бесплатного мессенджера для улучшения связи между потребителем и производителем.

[2.1. Требования к аппаратному обеспечению](#content)<a id="environment1"></a> 
========================================
Требования к аппаратному обеспечению для юзера веб-сайта на компьютере:

* Процессор: Intel Core i3 или аналогичный
* Оперативная память: 4 Гб
* Видеокарта: совместимая с DirectX 11 и более поздней версии
* Жесткий диск: 250 Гб
* Разрешение экрана: 1024 х 768 дюймов и выше

Требования к аппаратному обеспечению для юзера веб-сайта на телефоне:

* Процессор: Apple A9 / Samsung Exynos 8890 или более поздняя версия процессора
* Оперативная память: 2 Гб для Android, 3 Гб для iOS
* Хранение: 16 Гб или более
* Разрешение экрана: 720 x 1280 пикселей или больше
* Интернет соединение: 3G или выше, Wi-Fi.

Это минимальные требования, чтобы гладко использовать веб-сайт на компьютере или телефоне. Если требования выше указанных, то пользователь сможет использовать сайт с еще большей производительностью.

[3. Функциональные требования](#content)<a id="functionality"></a>
========================================
Система должна позволять **Администратору**:
- Сервис должен позволять размещать в своей базе данных новых пользователей
- Сервис должен позволять размещать на своей базе данных новые объявления
- Сервис должен позволять редактировать в своей базе данных уже имеющихся пользователей
- Сервис должен позволять редактировать в своей базе данных уже имеющиеся объявления
- Сервис должен позволять удалять в своей базе данных объявления
- Сервис должен позволять удалять в своей базе данных данные пользователя



Система должна позволять **Потребителю**:
- Сервис должен позволять искать объявления в каталоге
- Сервис должен позволять оставлять отзывы о работе производителя
- При нажатии кнопки "Фильтр" система должна показать всплывающее окно перед поисковой системой с полями для заполнения
- При заполнении полей фильтрации, поисковая система должна использовать каждое условие, использованное в фильтре
- При заполнении полей фильтрации, поисковая система должна отменять каждое условие поиска, если пользователь его отменил
- При заполнении полей фильтрации, поисковая система должна показать все услуги, которые соответствуют поисковому запросу пользователя
- При заплднении полей фильтрации, поисковая система должна показать странницу с уведомлением о том, что данной услуги не существует, если система не находит её
- При нажатии строки с надписью "Поиск" пользователю должен быть выдан доступ к печати символов в строке поиска
- Поисковая система должна показывать пользователю предположительные услуги в зависимости от напечатанных символах в поисковой строке
- Поисковая система должна показать странницу с уведомлением о том, что данной услуги не существует, если система не находит её



Система должна позволять **Производителю**:
- При нажатии кнопки "Создать объявление" пользователь должен переадресовываться на странницу с формой создания объявления
- Сервис должен позволять настраивать тип объявления предоставляемых услуг
- При заполнении формы создания объявления система должна выделить обязательные поля с информацией, заполняющиеся пользователем
- При заполнении формы создания объявления система должна ярко выразить незаполненные поля с информацией и не дать доступ к созданию объявления, если такие присутствуют
- При заполнении формы создания объявления система должна позволять пользователю выбирать категорию услуги
- При заполнении формы создания объявления система должна позволить пользователю выбрать категорию услуги "Другое", если в перечне не присутствуют необходимая
- При заполнении формы создания объявления система должна показать пользователю необходимые поля для заполнения в зависимости от вида услуги
- При заполнении формы создания объявления система должна позволять загрузку фото и видео на усмотрения пользователя
- При заполнении формы создания объявления система не должна создавать новое объявление, если есть обязательные незаполненные поля
- При заполнении формы создания объявления система не должна создавать новое объявление, если поле для заполнения содержит неподдерживаемые символы
- При корректном заполнении формы создания объявления система должна создать новое объявление со всей имеющейся информацией, которую вводил пользователь и переадресовывать пользователя на странницу с созданным объявлением
- При нажатии на кнопку "Удалить объявление" система должна показывать всплывающее окно с подтверждением удаления объявления
- При нажатии на кнопку "Да" в всплывающем окне, система должна снять стату объявление "Активное" и убирает его в "Архив" с возможностью восстановить объявление
- При нажатии на кнопку "Восстановить объявление" система должна переадресовывать пользователя на странницу с заполнением формы объявление со всеми уже заполненными данными



**Общие возможности**:
- При нажатии кнопки "Заблокировать", система должна запретить пользователю отправлять и принимать сообщения от заблокированного пользователя
- При нажатии кнопки "Разблокировать", система должна снять запрет пользователю на отправление и принятие сообщений от разблокированного пользователя
- При нажатии на пользователя в диалоговом окне система должна открывать всплывающее окно с функциями "Закрепить диалоговое окно", "Пожаловаться", "Заблокировать" и "Перейти к профилю"
- При нажатии на кнопку "Пожаловаться" система должна открывать всплывающее окно с полями для заполнения информации о нарушении пользователя
- При нажатии кнопки "Отправить", система должна отправить обращение к администрации сервиса для рассмотрения
- При нажатии на кнопку "Закрепить диалоговое окно" система должна закрепить диалоговое окно с пользователем в самом верху списка
- При нажатии на кнопку "Открепить диалоговое окно" система должна откреплять диалоговое окно с пользователем в самом верху
- При нажатии на кнопку "Мои сообщения" система должна переадресовывать пользователя на странницу с диалоговыми окнами
- Если в истории пользователя нету сообщений, то странница должна вывести на экран уведомление об отстутствии диалоговых окон
- При нажатии на диалоговое окно с продавцом или покупателем система должна переадресовывать пользователя на странницу с мессенджерем
- При нажатии на объявление пользователь должен переадресовываться на странницу созданного объявления
- При нажатии на фотографию или видео на страннице объявления система должна вывести фотографию или видео на полный экран в полном разрешении
- При нажатии кнопки "Вход" пользователь должен переадресовываться на странницу с формой входа на сайт
- При некорректном запполнении формы входа система должна уведомить о неправильном написании логина и пароля и не дать доступ к личной страннице пользователя
- При корректном заполнении формы входа система должна дать доступ к личной страннице пользователя
- При нажатии кнопки "Регистрация" пользователь должен переадресовываться на странницу с формой регистрации на сайт
- При заполнении формы регистрации система должна выделить обязательные поля с информацией, заполняющиеся пользователем
- При заполнении формы регистрации система не должна дать пользователю зарегистрироваться и должна ярко выразить незаполненное поле, если такое присутствует
- При заполнении формы регистрации система должна не дать пользователю зарегистрироваться, если в поле для заполнения используются неподдерживаемые символы
- При заполнении формы регистрации в графе "Пароль" система должна указывать надёжность пароля пользователя
- При заполнении формы регистрации система должна показать пользователю необходимые поля для заполнения для "Производителя" или "Потребителя"
- При корректном заполнении формы регистрации система должна создать новую учётную запись со всей имеющейся информацией, которую вводил пользователь
- При некорректном заполнении формы регистрации система должна показать пользователю неправильно заполненные поля и не создавать новую учётную запись
- При нажатии на кнопку "Перейти к профилю" система должна переадресовывать пользователя на странницу другого пользователя
- На страннице пользователя должна указываться средняя оценка работы пользователя, которая вычисляется средне арифметически
- На страннице пользователя должны показываться все объявления созднанные пользователем
- На страннице пользователя должна быть информация об отсутствии объявлений, если их не существует
